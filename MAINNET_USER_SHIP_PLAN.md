# Mainnet User Ship Plan

This is the shortest path from demoable product to real users.

## Product To Sell First

Start with one paid AI/SaaS launch:

- Founder sells a credit pack, support agent, API access, template, or micro-SaaS subscription.
- Buyer pays through Dodo checkout.
- DodoLaunch records the paid event.
- Founder reviews revenue splits.
- Wallet operator approves USDC payout only when ready.

Do not start by supporting every possible SaaS model. Win the first real sale flow first.

## Mainnet Readiness

- Connect Phantom in the app.
- Add live Dodo product ID and live Dodo API key in Vercel.
- Add webhook secret in Vercel.
- Fund the operator wallet with enough SOL for fees.
- Fund the source USDC token account only for the batch being tested.
- Run one tiny real payout before sending to users.

## First Users

Target users who already sell something digital:

- AI automation freelancers.
- Micro-SaaS builders.
- Paid API builders.
- Template or tool sellers.
- Small agencies paying affiliates, vendors, or contributors.

## What To Prove

- A real buyer can reach checkout.
- A paid event lands in the ledger.
- Split math is correct.
- CSV export is useful for ops.
- A wallet-approved USDC payout can produce a real Solscan signature.

## Do Not Build Yet

- Full login system.
- Hosted database.
- Team roles.
- Multi-chain support.
- Complex analytics.
- AI API features that cost money.

Build these only after users say the current workflow is worth keeping.
